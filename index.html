Introdução a Banco de Dados

Um banco de dados relacional é uma coleção organizada de informações estruturadas em tabelas, compostas por linhas e colunas.

As tabelas representam entidades do mundo real, e as linhas correspondem aos registros ou instâncias dessas entidades. Cada coluna contém um tipo de dado específico, como texto, número ou data.

A ideia principal dos bancos de dados relacionais é estabelecer relações entre as tabelas por meio de chaves primárias e estrangeiras. Uma chave primária é um identificador único para cada registro em uma tabela, enquanto uma chave estrangeira é uma referência a uma chave primária de outra tabela.

Essas relações permitem a realização de consultas e junções de dados, facilitando a busca, a recuperação e a manipulação das informações de forma eficiente. Além disso, os bancos de dados relacionais seguem os princípios da integridade referencial, garantindo a consistência dos dados.

É importante entender os conceitos básicos de banco de dados relacional para projetar esquemas eficazes, normalizar as tabelas adequadamente e escrever consultas eficientes. Isso permite um armazenamento e recuperação de dados organizados e confiáveis para aplicações e sistemas.


Modelo Entidade-Relacionamento

O modelo entidade-relacionamento (modelo ER) é uma forma de representar dados em um banco de dados. É um modelo gráfico que mostra as entidades (objetos), seus atributos (propriedades) e os relacionamentos entre eles. Os modelos ER são usados para projetar bancos de dados antes de serem criados.

Os elementos básicos de um modelo ER são:

  • Entidades: As entidades são os objetos armazenados em um banco de dados. Podem ser pessoas, lugares, coisas ou conceitos.
  • Atributos: Os atributos são as propriedades das entidades. Descrevem as entidades com mais detalhes. Por exemplo, uma entidade pessoa pode ter atributos como nome, idade e endereço.
  • Relacionamentos: Os relacionamentos são as conexões entre as entidades. Mostram como as entidades estão relacionadas entre si. Por exemplo, uma entidade pessoa pode ter um relacionamento com uma entidade carro, mostrando que a pessoa é proprietária do carro.

Os modelos ER são usados para projetar bancos de dados porque oferecem uma forma clara e concisa de representar dados. Podem ser usados para se comunicar com projetistas de bancos de dados, desenvolvedores e usuários.

Há dois tipos principais de modelos ER:

  • Modelos ER básicos: Os modelos ER básicos são o tipo mais simples de modelo ER. Eles mostram apenas entidades, atributos e relacionamentos.
  • Modelos ER aprimorados: Os modelos ER aprimorados são mais complexos que os modelos ER básicos. Podem mostrar informações adicionais, como cardinalidades e restrições.

As cardinalidades são usadas para mostrar o número de relacionamentos que podem existir entre duas entidades. Por exemplo, um relacionamento um-para-um significa que cada entidade pode estar relacionada a apenas uma outra entidade. Um relacionamento um-para-muitos significa que cada entidade pode estar relacionada a várias outras entidades. Um relacionamento muitos-para-muitos significa que cada entidade pode estar relacionada a várias outras entidades, e cada outra entidade pode estar relacionada a várias entidades.

As restrições são usadas para mostrar regras que devem ser seguidas pelos dados no banco de dados. Por exemplo, uma restrição pode dizer que o valor de um atributo deve ser um número.

Os modelos ER são uma ferramenta poderosa para o projeto de bancos de dados. Podem ser usados para criar representações claras e concisas de dados e podem ser usados para se comunicar com projetistas de bancos de dados, desenvolvedores e usuários.

Modelo Relacional

O modelo relacional é uma forma de representar dados em um banco de dados. É um modelo lógico, o que significa que não depende de uma implementação física específica. O modelo relacional é baseado na ideia de que os dados podem ser representados como uma coleção de tabelas, cada uma contendo um conjunto de dados relacionados.

Os elementos básicos de um modelo relacional são:

  • Tabelas: As tabelas são a unidade básica de armazenamento de dados em um banco de dados relacional. Uma tabela consiste em um conjunto de linhas e colunas. Cada linha representa um único registro e cada coluna representa um único atributo desse registro.
  • Colunas: As colunas são as divisões verticais de uma tabela. Cada coluna representa um único atributo dos dados armazenados na tabela.
  • Linhas: As linhas são as divisões horizontais de uma tabela. Cada linha representa um único registro de dados armazenados na tabela.
  • Chaves primárias: As chaves primárias são colunas que identificam de forma única cada registro em uma tabela.
  • Chaves estrangeiras: As chaves estrangeiras são colunas que se referem à chave primária de outra tabela.
  • Restrições: As restrições são regras que limitam os dados que podem ser armazenados em uma tabela.

O modelo relacional é uma ferramenta poderosa para modelagem de dados e projeto de banco de dados. É fácil de entender e usar, e é suportado pela maioria dos principais sistemas de gerenciamento de banco de dados.

Aqui estão alguns dos benefícios de usar o modelo relacional:

  • Integridade dos dados: O modelo relacional impõe a integridade dos dados usando restrições. Isso ajuda a garantir que os dados em um banco de dados sejam precisos e consistentes.
  • Independência dos dados: O modelo relacional é baseado no conceito de independência dos dados, o que significa que os dados em um banco de dados podem ser alterados sem afetar os programas de aplicação que o utilizam. Isso facilita as alterações no banco de dados sem ter que reescrever os programas de aplicação.
  • Segurança dos dados: O modelo relacional pode ser usado para implementar a segurança dos dados, controlando quem tem acesso aos dados e o que eles podem fazer com eles.

O modelo relacional é um modelo de dados amplamente aceito e utilizado. É a base da maioria dos sistemas modernos de gerenciamento de banco de dados.


Normalização

A normalização é um processo de organização de dados em um banco de dados para reduzir a redundância e melhorar a integridade dos dados. É uma forma de estruturar um banco de dados de maneira lógica e eficiente.

A normalização foi desenvolvida pela primeira vez por Edgar F. Codd na década de 1970. Codd foi um cientista da computação britânico considerado o pai do modelo de banco de dados relacional.

A normalização é necessária por várias razões:

  • Para reduzir a redundância: A redundância ocorre quando os mesmos dados são armazenados em vários locais em um banco de dados. Isso pode levar a erros e inconsistências, além de dificultar a atualização dos dados.
  • Para melhorar a integridade dos dados: A normalização ajuda a garantir que os dados em um banco de dados sejam precisos e consistentes. Isso ocorre porque a normalização elimina dados redundantes, o que pode levar a erros.
  • Para melhorar o desempenho: A normalização pode melhorar o desempenho de um banco de dados reduzindo a quantidade de dados que precisam ser acessados.

Existem três tipos de anomalias que podem ocorrer em um banco de dados:

  • Anomalia de inserção: Isso ocorre quando um novo registro não pode ser inserido em uma tabela porque os dados desse registro já estão armazenados em outra tabela.
  • Anomalia de atualização: Isso ocorre quando uma alteração em um valor em uma tabela não é refletida em outra tabela.
  • Anomalia de exclusão: Isso ocorre quando um registro é excluído de uma tabela e os dados desse registro também são excluídos de outras tabelas.

Processo de Normalização

A normalização é um processo de conversão de um banco de dados de uma forma menos normalizada para uma forma mais normalizada. Existem três níveis principais de normalização:

  • Primeira forma normal (1NF): Uma tabela está na 1NF se não contiver nenhum dado duplicado.
  • Segunda forma normal (2NF): Uma tabela está na 2NF se estiver na 1NF e todas as colunas não chave são totalmente dependentes da chave primária.
  • Terceira forma normal (3NF): Uma tabela está na 3NF se estiver na 2NF e não houver dependências transitivas.

Etapas do Processo de Normalização

  1. Refatoração: A primeira etapa é refatorar o banco de dados, identificando e removendo quaisquer dados duplicados.
  2. Determinação: A segunda etapa é determinar a chave primária de cada tabela.
  3. Dependência Funcional: A terceira etapa é identificar as dependências funcionais entre as colunas de cada tabela.
  4. Classificação das Dependências Funcionais: A quarta etapa é classificar as dependências funcionais em três tipos: triviais, parciais e total
  5. Diagrama de Dependência:

    6. Um diagrama de dependência é uma representação gráfica das dependências funcionais entre as colunas de uma tabela.

    Passagem para a Primeira Forma Normal (1NF)

    Uma tabela está na 1NF se não contiver nenhum dado duplicado. Para converter uma tabela para a 1NF, todos os dados duplicados devem ser removidos.

    Passagem para a Segunda Forma Normal (2NF)

    Uma tabela está na 2NF se estiver na 1NF e todas as colunas não chave forem totalmente dependentes da chave primária. Para converter uma tabela para a 2NF, todas as colunas não chave que não são totalmente dependentes da chave primária devem ser removidas.

    Passagem para a Terceira Forma Normal (3NF)

    Uma tabela está na 3NF se estiver na 2NF e não houver dependências transitivas. Para converter uma tabela para a 3NF, todas as dependências transitivas devem ser removidas.

    Desnormalização:

    A desnormalização é o processo de reverter a normalização. A desnormalização às vezes é feita para melhorar o desempenho de um banco de dados. No entanto, a desnormalização também pode levar a redundância e inconsistência de dados.

    Comandos básicos de SQL para criar, excluir, pesquisar e juntar. DDL e DML

    A seguir estão alguns comandos básicos de SQL que podem ser usados para criar, excluir, pesquisar e juntar tabelas.

    Criação

    O comando `CREATE` é usado para criar uma nova tabela em um banco de dados. A sintaxe para o comando `CREATE` é a seguinte:

    CREATE TABLE table_name (column_name data_type, column_name data_type, ...)

    Por exemplo, o seguinte comando cria uma tabela chamada clientes com três colunas: customer_id, customer_name e customer_email:

    CREATE TABLE customers (customer_id INT, customer_name VARCHAR(255), customer_email VARCHAR(255))

    Inserir

    O comando INSERT é usado para inserir uma nova linha em uma tabela. A sintaxe para o comando INSERT é a seguinte:

    INSERT INTO nome_da_tabela (nome_da_coluna, nome_da_coluna, ...) VALUES (valor, valor, ...)

    Por exemplo, o seguinte comando insere uma nova linha na tabela de clientes com os seguintes valores:

    customer_id = 1 customer_name = "John Doe" customer_email = "johndoe@example.com"

    INSERT INTO customers (customer_id, customer_name, customer_email) VALUES (1, "John Doe", "johndoe@example.com")

    O comando INSERT também pode ser usado para inserir várias linhas em uma tabela de uma só vez. Para fazer isso, basta separar as linhas com uma vírgula. Por exemplo, o seguinte comando insere duas novas linhas na tabela de clientes:

    customer_id = 1, customer_name = "John Doe", customer_email = "johndoe@example.com" customer_id = 2, customer_name = "Jane Doe", customer_email = "janedoe@example.com"

    INSERT INTO customers (customer_id, customer_name, customer_email) VALUES (1, "John Doe", "johndoe@example.com"), (2, "Jane Doe", "janedoe@example.com")

    Excluir

    O comando DELETE é usado para excluir linhas de uma tabela. A sintaxe para o comando DELETE é a seguinte:

    DELETE FROM table_name WHERE condition

    Por exemplo, o comando a seguir exclui todas as linhas da tabela de clientes em que customer_id é igual a 1:

    DELETE FROM customers WHERE customer_id = 1

    Pesquisar

    O comando SELECT é usado para selecionar linhas de uma tabela. A sintaxe para o comando SELECT é a seguinte:

    SELECT column_name, column_name, ... FROM table_name WHERE condition

    Por exemplo, o comando a seguir seleciona todas as linhas da tabela de clientes em que customer_name é igual a "John Doe":

    SELECT customer_name FROM customers WHERE customer_name = "John Doe"

    Junção

    O comando JOIN é usado para combinar linhas de duas ou mais tabelas. A sintaxe para o comando JOIN é a seguinte:

    SELECT nome_coluna, nome_coluna, ... FROM nome_tabela1 JOIN nome_tabela2 ON condição

    Por exemplo, o seguinte comando une a tabela customers com a tabela orders na coluna customer_id. Isso retornará todas as linhas da tabela customers, juntamente com as linhas correspondentes da tabela orders:

    SELECT customer_name, customer_email, order_id, order_date FROM customers JOIN orders ON customers.customer_id = orders.customer_id

    DDL e DML

    DDL e DML são dois tipos de comandos SQL. DDL significa Linguagem de Definição de Dados, e é usada para criar, alterar e excluir tabelas. DML significa Linguagem de Manipulação de Dados, e é usada para inserir, atualizar e excluir linhas de tabelas.

    Aqui está uma tabela que resume os diferentes tipos de comandos SQL:

    Comando | Descrição
    ---|---
    `CREATE` | Cria uma nova tabela.
    `ALTER` | Altera uma tabela existente.
    `DROP` | Exclui uma tabela.
    `INSERT` | Insere uma nova linha em uma tabela.
    `UPDATE` | Atualiza uma linha existente em uma tabela.
    `DELETE` | Exclui uma linha de uma tabela.
    `SELECT` | Seleciona linhas de uma tabela.
    `JOIN` | Combina linhas de duas ou mais tabelas.
